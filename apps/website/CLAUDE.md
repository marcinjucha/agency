# apps/website/ - Public Website

Public-facing marketing website and client survey forms.

## Purpose

Public application for:
- Marketing agency services
- Client survey form submission (via unique token links)
- Calendar appointment booking
- No authentication required

## Tech Stack

- **Framework:** TanStack Start v1.167 + Vite 8
- **Forms:** React Hook Form + Zod
- **UI:** shadcn/ui from @agency/ui
- **Database:** Supabase (public access via RLS)
- **State:** No complex state management needed (simple forms)

## Key Concepts

### No Authentication
This app is completely public. No login, no middleware, no protected routes.

### Survey Flow
1. Client receives email with link: `agency.com/survey/abc123`
2. Clicks link в†’ opens survey form
3. Fills out form в†’ submits
4. Sees calendar в†’ books appointment
5. Gets confirmation

### Data Fetching Pattern (TanStack Start)
Routes use `createServerFn` (in `features/*/server.ts`) called from route `loader`.
Component reads data via `Route.useLoaderData()`.

```typescript
// features/marketing/server.ts
export const getPublicLandingPageFn = createServerFn().handler(async () => { ... })

// app/routes/index.tsx
export const Route = createFileRoute('/')({
  loader: async () => ({ page: await getPublicLandingPageFn() }),
  component: function HomePage() {
    const { page } = Route.useLoaderData()
    return <LandingPage page={page} />
  },
})
```

### Internal Links: `<Link>` not `<a>`
Always use `@tanstack/react-router` `<Link>` for same-domain routes. Plain `<a>` only for external URLs.

## Database Access

**`createServiceClient()` for all server queries** вЂ” in `lib/supabase/service.ts`. Service role, no cookies required.

**Why:** TanStack `createServerFn` runs server-only. No cookie context needed. Service role is safe because website only reads public data (RLS on `is_published`) or writes submissions (tenant_id comes from survey row, not user input).

## State Management

**No TanStack Query** вЂ” TanStack Query is CMS-only. Website uses `Route.useLoaderData()` from the route loader.

**React Hook Form** for survey forms.

## Routes

```
/                          - Homepage (landing page from CMS)
/blog                      - Blog listing
/blog/[slug]               - Blog post
/survey/[token]            - Client survey form + calendar booking
/regulamin                 - Terms of service
/polityka-prywatnosci      - Privacy policy
```

## Survey + Booking Flow

Survey submission uses `createServerFn` (not API Route). The server function uses `createServiceClient()` (service role) which does not depend on cookies.

**CalendarBooking** split into sub-components: `DateSlotPicker` (owns slot-fetch state, accepts `surveyId` prop), `BookingForm`, `TimeSlotsGrid`, `BookingSuccess`. **Why:** Original monolithic component was 400+ lines mixing 4 concerns.

## Deployment

Auto-deploys to: https://agency-website.vercel.app

**Environment Variables (Vercel):**
- `VITE_SUPABASE_URL` (public Supabase URL, safe for client bundle)
- `SUPABASE_SERVICE_ROLE_KEY` (server-only, for createServiceClient)
- `N8N_WEBHOOK_SURVEY_ANALYSIS_URL` (server-only, AI analysis)
- `N8N_WEBHOOK_FORM_CONFIRM_EMAIL_URL` (server-only, email confirmation)
- `CMS_BASE_URL` (server-only, workflow trigger API)
- `WORKFLOW_TRIGGER_SECRET` (server-only, workflow auth)
