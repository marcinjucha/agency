  • Notes: 0-500 characters (Zod, optional)
  • Validation Mode: onBlur (validates on field blur)

API ENDPOINTS:
  1. GET /api/calendar/slots?surveyId={id}&date={YYYY-MM-DD}
     → Returns: { slots: [{ start, end }] }
     → Errors: 404 (not connected), 500 (server error)

  2. POST /api/calendar/book
     → Body: surveyId, responseId, startTime, endTime,
              clientName, clientEmail, notes, lawyerEmail
     → Response: { success: boolean, error?: string }
